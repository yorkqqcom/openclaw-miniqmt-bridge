## 核心叙事弧线：从裸脚本到工程化（建议放在销售材料最前）

不要从功能列表开场；用**一条演进路径**让不同角色立刻对号入座。PPT / 官网首屏可用标题：**「从裸脚本到工程化：一套典型演进路径」**。

### 三步递进（痛点 → 价值）

1. **起点：信号与下单缠在一起**  
   策略写在 Jupyter/脚本里，下单与行情回调散落各处；半夜担心网络抖动、重复点击或逻辑分支导致**重复下单**，却难以复盘「当时到底是谁下的单」。

2. **转折：策略只负责「想什么」**  
   策略只输出一个简单的 **`OrderIntent` / `TargetPosition`**；是否下单、如何幂等、如何过风控单笔上限、如何跟券商回报对齐，全部交给**执行中间件**。

3. **结果：可审计、可观测、可交付**  
   统一 HTTP API 与管理 UI 看资金与委托；Prometheus 与日志服务运维；需要 AI 辅助时走 **MCP** 工具链；培训或商业化时由流程与合同约束装机与期限。你从「脚本能跑」升级到「系统能运营」。

### 角色对号入座（口语化摘要）

- **个人量化者**：「你从 Jupyter 脚本起步，策略信号和下单代码缠在一起，半夜惊醒担心重复下单。现在你把策略输出一个简单的 OrderIntent，剩下的编排、风控、对账全部交给这个中间件。」

- **培训 / 社群运营者**：「你不再需要追着学员收安装包，也不用担心课程结束后环境失控。统一装机包 + 管理台账户档案，按清单回收即可。」

### MCP：为什么是现在（一句话放进定位页，详细口播见 `docs/demo_assets.md`）

2025 年起，Cursor、Windsurf 等 AI 编程助手已普遍支持 **MCP** 接入外部工具。本系统将账户查询、历史与行情类能力、以及受 **`TRADING_ENABLED`** 与风控约束的下单路径封装为标准 MCP 工具——**不是代替你决策**，而是在你设定的权限边界内，用自然语言与自动化节省操作与核对时间。

- 安全交付：**API Token、会话鉴权、指标与部署手册，便于单机或反代上线。**

## 我们是什么 / 不是什么

| 我们是什么 | 我们不是什么 |
|------------|--------------|
| 执行与运营层软件（下单编排、查询、历史、策略看板、Agent 工具入口） | 不承诺超额收益或「稳赚」策略 |
| 与 miniQMT/xtquant 适配的工程化骨架 | 不替代券商交易终端、交易所规则或适当性管理 |
| 可集成到自有策略与风控流程的组件 | 不提供具体证券买卖建议或代客理财 |

## 我们刻意不做的事（对外可单独成版：信任信号）

专业团队要的是**边界清晰的可控工具**，而不是黑盒圣杯。以下可原样或微调用于官网底部、销售首页独立板块标题**「我们刻意不做的事」**：

- **不做收益承诺**：不宣传年化、回撤、胜率或「稳赚」；策略表现由你的模型与市场决定。
- **不碰你的策略逻辑**：不在产品层替你选股或生成买卖建议；我们承接的是 **OrderIntent 之后的执行与状态管理**。
- **不替代券商终端与监管义务**：适当性、交易权限、交易所规则仍以券商与监管为准；我们是通道之上的编排与观测层。

对应话术（可上官网）：

> 我们刻意不做收益承诺、不替代券商终端、不在产品内替你写策略逻辑。正是因为边界清晰，你才能放心把**执行层**交给我们——我们只做**下单编排与状态管理**这一件事，并把它做到可审计、可观测。

## 推荐用语（可写进官网/白皮书）

- 强调：**可观测性**（日志、Prometheus）、**幂等与对账**、**联调开关**（`TRADING_ENABLED`）、**文档与运行手册**。
- 强调：**策略层与执行层分离**（`OrderIntent` / `TargetPosition`），便于审计与迭代。
- 强调：**MCP** 在受控边界内充当「交易副驾驶」：查询、核对、辅助流程；写操作仍受环境与风控闸门约束（见 `docs/demo_assets.md` 分镜与口播）。

## 禁宣边界（收益与合规：内部清单）

以下内容**不建议**在公开宣传、销售话术或合同中由卖方主动承诺；若客户要求写入，须法务评估。对外不必以「禁止列表」形式展示，可改用上一节 **「我们刻意不做的事」** 的正向表述。

1. **收益类**：年化收益、回撤上限、胜率、「跟单必赚」等表述。
2. **监管越界**：暗示本产品已完成某类金融业务许可或等效于持牌投顾/资管。
3. **全责任转嫁**：如「使用本系统产生的亏损由开发商承担」等不可保真条款。
4. **夸大范围**：在未给出环境矩阵的前提下宣称「全市场、全品种、全券商无条件支持」。

## 建议的免责声明要素（示例，须本地化审阅）

- 软件为**交易执行与信息管理工具**；证券交易风险由用户自行承担。
- 实盘行为须遵守**交易所、券商及当地监管**规则；系统内风控参数（如单笔名义金额上限）为**软件侧约束**，与券商额度等无关。
- 行情、委托状态以**上游 QMT/xtquant 及券商**为准；文档中的 A 股规则表格多为教学参考，**不构成合规结论**。
