# 微信播报 / 私域短文本输出规范

**适用范围：** 微信对话、私域短讯中的「人读」摘要。**不**适用于钉钉、邮件、PDF 报告；其他渠道请另建规范文件。

## 规则摘要

- 每行不超过 **25 个汉字**。  
- 分隔线使用 `━━`，避免过长 Unicode 框线。  
- 数字优先用 **万** 等单位（如 990 万），避免长串逗号分隔。  
- 尽量避免 Markdown **表格**与复杂框线字符（如 `╔║╚═`）；必要时用缩进列表。  
- 每只标的尽量 **一行** 展示；emoji 紧贴语义（如 `⚠️` 大幅流出、`⭐` 正增长）。

**影响：** 不满足时消息在微信中折行难看，且超出屏幕宽度。**详细示例**见各 skill 中「输出格式」小节；真源为 `signal-trader` / `closing-review` 内模板时，**不要在本文件重复粘贴大段样例**，以免双份漂移。

**消费方：** `daily-morning-check`、`closing-review`、`signal-trader`（计划与复盘输出）。

**修改本文件后：** 同步检查上述 skill 是否仍写死旧规则（如仍写「25 字」但与本文件矛盾）。
